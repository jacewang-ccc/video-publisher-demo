# PRD v1.0 (MVP) — 抖音优先，可扩展多平台

## 1. 一句话
做一个「网页统一面板 + 浏览器扩展编排器」：你上传一次视频，在网页里看到**所有平台最终要发布的字段文本**并确认；确认后扩展自动打开各平台发布页并完成“准备就绪”；你在网页点一次“最终发布”，扩展对所有平台执行发布。

> 约束：自动修正优先；发布前强制确认；所有规则/模板/分类/合集/排期可视化配置、可版本化、可回滚。

## 2. MVP 范围

### 做（第一期）
- 单人版（无团队/权限）
- 抖音平台：连接器可完成 Prepare + Commit（可先以 stub/半自动落地，逐步增强）
- 网页统一面板（静态即可）：内容创建、生成、预览矩阵、锁定快照、发布任务状态、配置中心
- 配置中心：分类树、模板包、规则、排期策略、Bundle 组合
- 约束/修正引擎：超限截断、tag 裁剪、定时窗口校验与自动调整、变更清单
- 两阶段发布（2-phase publish）：
  - Prepare：锁定后扩展后台准备各平台到“可发布状态”
  - Commit：网页一键最终发布

### 不做（先不承诺）
- 完全无人值守绕过验证码/风控
- 数据报表（播放/互动）
- 多人协作

## 3. 关键用户偏好（默认配置，可改）
- 抖音默认：公开
- 抖音默认：不允许保存
- 拍摄地点：默认深圳（可改）
- 拍摄日期：默认当天（可改）
- 发布时间：
  - 你每次手动选“基准日期”（默认今天）
  - 抖音：基准日期 20:00
  - 小红书：次日 10:00（预留）
  - 视频号：次日 12:00（预留）
  - B站：次日 18:00（预留）
- 副线子类（初始）：书页时光；AI方法论/AI工具（后续可新增）

## 4. 内容画像（不写死，可扩展）
- 分类树（taxonomy）：主类 → 子类 → …（无限层）
  - 预置：日更/副线/主线
  - 日更预置 6 主题：表达/唱歌/冥想/阅读/运动/英语口语
- 标签（labels）：自由标签（如：讲书、含AI片段、碎碎念、深圳、系列A等）
- 变量（vars）：可选字段（打卡天数 day_n、主题 daily_theme、书名 book_name…）

## 5. 自动化模块体系（可组合）

每个模块都声明：
- Scope：适用平台（global/抖音/平台集合）、适用字段（title/desc/tags/…）、适用内容范围（分类路径/标签）
- Slot：执行阶段（extract/generate/transform/validate/repair/finalize/execute）
- IO：输入输出（统一 Draft + ChangeLog）

通过 Bundle（组合包）把多个细粒度模块组装成“一键方案”。

## 6. 两阶段发布（你要的一次确认 + 一键发布）

### 6.1 Prepare：预览矩阵 + 锁定快照
- 网页展示多平台预览矩阵（每平台一行）：
  - 发布时间、标题、简介、话题、tag、合集、声明（含地点日期）、发布设置
  - 校验状态：通过/已自动修正/需处理（阻止锁定）
  - 自动修正清单（可展开）
- 你点「锁定并准备发布」后：
  - 生成 snapshot（不可变快照）并记录配置版本
  - 扩展后台打开各平台发布页并完成字段填充/上传/设置，状态回传网页

### 6.2 Commit：一键最终发布
- 网页点一次「最终发布（全部平台）」：
  - 扩展对所有“已就绪”的平台自动点击发布
  - 对“待扫码/验证码”的平台暂停并提示你处理，处理后继续

## 7. 页面清单（MVP）
- 内容库
- 创建/编辑内容
- 预览矩阵（Prepare & Lock）
- 发布任务（Prepare 状态、Commit 状态、重试）
- 配置中心（分类/模板/规则/排期/Bundle）

## 8. 验收标准（第一期）
- 能在网页里生成抖音最终字段文本，并看到“自动修正清单”
- 锁定后产生 snapshot；扩展能收到 snapshot 并进入 Prepare，网页显示状态
- 网页点一次“最终发布”可触发扩展 Commit（至少在 demo/stub 环境完成闭环）
- 能在配置中心新增一个子类并配置模板；下一条内容可选择并生效
